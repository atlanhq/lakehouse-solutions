# Setup Guide: Atlan Lakehouse on GCP (GCS)

This guide covers the cloud-specific setup steps for connecting your **GCP Databricks** workspace to Atlan Lakehouse data stored in **Google Cloud Storage (GCS)**.

> **Note:** This setup involves a back-and-forth with Atlan Support. You will create a storage credential, share the Databricks-managed service account email with Atlan, and wait for Atlan to confirm before proceeding.

## What Atlan Provides

After you contact Atlan Support (see [README](README.md#2-contact-atlan-support)), Atlan will provide:

- **GCS bucket path** (e.g., `gs://<bucket-name>/<catalog-name>`)
- **Catalog details** (Catalog URI, Catalog Name, Warehouse Name)
- **OAuth credentials** (Client ID and Client Secret)

## Step 1: Create a Storage Credential

Create a [storage credential](https://docs.databricks.com/gcp/en/connect/unity-catalog/cloud-storage/storage-credentials) in Unity Catalog using a Google Cloud service account.

1. Navigate to **Catalog Explorer > Credentials > Create Credential**
2. Select **GCP Service Account** as the credential type
3. Provide a credential name
4. In **Advanced Options**, enable **Limit to read-only use**
5. Click **Create**

Databricks will automatically generate a managed Google Cloud service account and display its email address (e.g., `db-<uuid>@<project>.iam.gserviceaccount.com`). Copy this email — you will share it with Atlan in the next step.

## Step 2: Share the Service Account Email with Atlan

After creating the credential, share the following with Atlan Support:

- The **service account email** generated by Databricks for your storage credential

Atlan will grant this service account read access (`roles/storage.objectViewer`) on the GCS bucket and confirm back when complete.

> **Important:** Do not proceed to running the notebooks until Atlan confirms the service account has been granted access. You can continue with Steps 3 and 4 below while waiting.

## Step 3: Create an External Location

Create an [external location](https://docs.databricks.com/gcp/en/connect/unity-catalog/cloud-storage/external-locations) in Unity Catalog that points to the GCS path provided by Atlan.

1. Navigate to **Catalog Explorer > External Locations > Create External Location**
2. Select **Manual** and choose **Google Cloud Storage** as the storage type
3. Enter the **GCS path** provided by Atlan (e.g., `gs://<bucket-name>/<catalog-name>`)
4. Select the credential created in Step 1
5. Enable **Read-only mode** in advanced options
6. Click **Create**

> **Expected behavior:** A "Permission Denied" error may appear during creation if Atlan has not yet granted the service account access. Click **Force Create**. The connection will work once Atlan completes their side of the setup (Step 2).

## Step 4: Create a Target Catalog

Create a new Unity Catalog (or use an existing one) where the foreign Iceberg tables will be registered. This is the catalog name you will configure as `DBX_CATALOG_NAME`.

1. Navigate to **Catalog > Create a Catalog**
2. Provide a catalog name and choose **Standard** type
3. For storage location, use a **customer-managed** storage location on your own GCP project (not Atlan-managed)

## Next Steps

Once Atlan confirms the service account has been granted access, return to the [main README](README.md#configuration) to configure and run the notebooks.
